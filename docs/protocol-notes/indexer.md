# indexer

## Summary
DeepBook Predict = expiry-based binary-option/prediction protocol on Sui, priced off an on-chain SVI vol surface (Black-Scholes binary pricing). The predict-indexer (crates/predict-indexer) is a thin sui-indexer-alt-framework pipeline: 20 Move event types from package 0xf5ea2b3749c65d6e56507cc35388719aadb28f9cab873696a2f8687f5c785138 (testnet) are BCS-decoded into 1:1 append-only Postgres tables (predict-schema crate), keyed by event_digest = tx_digest||event_index, inserted with on_conflict_do_nothing. Units: prices/probabilities/SVI params in FLOAT_SCALING 1e9; quantities in 6-decimal quote units (1_000_000 = 1 contract = $1, winner receives quantity); signed SVI params (rho, m) are (magnitude u64, is_negative bool) pairs. Public read surface = https://predict-server.testnet.mystenlabs.com (full REST endpoint list in packages/predict/README.md); Predict shared object 0xc8736204d12f0a7277c86388a68bf8a194b0a14c5538ad13f22cbd8e2a38028a; quote asset DUSDC. Positions/ranges are NOT objects — they are balances inside the per-user shared PredictManager, keyed by MarketKey(oracle_id, expiry, strike, is_up) / RangeKey(oracle_id, expiry, lower_strike, higher_strike). Entry points for PTBs: predict::create_manager, mint<Quote>, redeem<Quote>, redeem_permissionless<Quote>, mint_range<Quote>, redeem_range<Quote>, supply<Quote>, withdraw<Quote>; manager funding via predict_manager::deposit. Quoting: bid/ask = oracle price ± (base_spread + skew + utilization), ask clamped to [1%, 99%] defaults, vault exposure cap 80%. The team's eng-rules files document Move style (macros over constants, leaf-level math guards, named abort codes, events past-tense), brutal unit-test discipline (never derive expected values from contract code, exact assert_eq only, scipy generate_constants.py as ground truth), and ops gotchas (SVI feed needs >=9 decimal subscription precision or 1e9 scaling flattens short-dated `a` to zero; oracle identity = (underlying, expiry); diesel migrations run in lexicographic order inside transactions).

### Oracle lifecycle
Before expiry: oracle must be ACTIVE to trade; prices update frequently, SVI less frequently; mints/redeems quote off current oracle state. At expiry: first post-expiry price push freezes settlement spot and emits OracleSettled. After settlement: positions/ranges redeemable, no further price/SVI updates accepted, vault may compact the settled oracle into constant-size settled state (internal optimization). Tradeability check = oracle status, not existence. Live-stream watch list: oracle::OraclePricesUpdated, OracleSVIUpdated, OracleSettled, OracleActivated filtered by current package id.

### Rules: move.md (team Move conventions)
Move 2024 edition, module-label style, prettier-move formatting. Prefer macros over constants (hence config defaults are macros). Error consts EPascalCase, regular ALL_CAPS, events past-tense, caps suffixed Cap, dynamic-field keys positional + Key suffix with use-fun aliases on UID. Function order: public -> public(package) -> private. Math/utility modules guard ONLY math preconditions (div-by-zero, overflow), never app policy; never let raw VM arithmetic errors leak — add named abort guards (EZeroForward, EZeroDivisor); never remove leaf underflow guards because 'caller validates' (cross-module invariant risk -> silent u64::MAX wraparound). Timestamp fields have single semantics (don't bump price-timestamp on SVI updates — breaks staleness checks). Validate before mutate. Test command needs --gas-limit 100000000000 (sui 1.66+ lowered default test gas budget).

### Rules: unit-tests.md (test discipline)
Born from the upstream tooling repeatedly writing tests confirming buggy behavior. Core rules: (1) never compute expected values via contract functions (circular); expected values from hand calc, generate_constants.py (scipy ground truth for ln/exp/normal_cdf/SVI/binary prices), Block Scholes CSVs in predict_research/, or reference impl; (2) never adjust expected values to make tests pass (Φ(0)=500_000_000 exactly, a 500_000_002 result is a real precision bug); (3) every test asserts outputs; (4) every E* abort code gets an expected_failure test with named code + trailing `abort 999` guard; (10) assert_eq! only, never approx/range — 1 unit of precision loss can be an exploit; (12) production-behavior tests must use production-valid fixtures (real strike-grid/tick constraints), permissive helpers must be named e.g. create_unbounded_test_oracle; (13) capture shared-object IDs and take_shared_by_id.

### Rules: scripts.md (ops/SVI feed gotchas)
(1) SVI subscriptions need >=9 decimals: BlockScholes model.params websocket formatting of 5 decimals quantizes into 1e-5 steps and can zero out `a` entirely for short-dated expiries because params are scaled 1e9 on-chain — keep precision >=9. (2) Oracle identity is (underlying, expiry), NOT (tier, expiry): inferTier() assigns one tier per oracle, so dedupe by expiry alone or overlapping 15m/1h schedules recreate the same expiry. (3) Lane gas-coin refs go stale ('Transaction needs to be rebuilt because object ... version ... is unavailable') — refresh gas object ref from chain or the lane retries a dead version and leaves expired oracles unsettled. ## Gotchas
- DB `timestamp` column is ingest-time (DEFAULT CURRENT_TIMESTAMP), never written by the indexer — for chain time always use checkpoint_timestamp_ms (checkpoint summary) or onchain_timestamp (the event's own Clock-derived u64 ms). Mixing them will skew the SVI surface time axis.
- Signed SVI params: rho and m arrive as I64{magnitude,is_negative} and are split into (rho, rho_negative)/(m, m_negative) BIGINT+BOOL column pairs — reconstruct sign before fitting/no-arb checks; a,b,sigma are unsigned. All SVI params are 1e9 fixed-point.
- All u64 amounts are cast `as i64` into BIGINT — values above i64::MAX would wrap silently (not realistic for prices, but don't assume unsigned semantics when reading the DB).
- Two different scales coexist: prices/strikes/SVI/spreads = 1e9 (FLOAT_SCALING), quantities/cost/payout/vault amounts = 1e6 quote units (6-decimal DUSDC). cost ≈ quantity * ask_price / 1e9. Winners get `quantity` directly at settlement (binary payout $1/contract).
- Positions/ranges are NOT objects — no NFT to query; read quantities from PredictManager object fields or the server endpoints. MarketKey/RangeKey are value keys constructed in the PTB, not object IDs.
- registry::OracleCreated has no predict_id field; the indexer infers it from the tx's 'Predict' input object and falls back to '0x0' — treat '0x0' rows as unresolved.
- event_digest is a plain string concat digest+event_index (e.g. 'txdigest5'), not a hash — fine as a dedup PK but don't parse it.
- Oracle settlement is push-driven: the first post-expiry price update settles; if the feed lane's gas coin ref goes stale, expired oracles can sit unsettled (documented ops failure mode). Don't assume settlement at exactly `expiry`.
- SVI updates are much less frequent than price updates — the surface viewer must carry forward the latest SVI per oracle and only re-time on OracleSVIUpdated; staleness threshold on-chain is 30s for prices.
- Ask prices are clamped to [min_ask_price, max_ask_price] (default 1%–99%, per-oracle overridable via OracleAskBoundsSet/Cleared) and include base+skew+utilization spread — naive Black-Scholes mid from the SVI surface will NOT equal executable ask/bid; the no-arb checker should compare against quoted ask/bid bounds, not mid.
- predict-server is low-lag but not zero-lag — after PTB execution, confirm via on-chain object reads, then refresh server endpoints (range-ladder executor must not poll the indexer for its own fill confirmation).
- Indexer depends on sui-indexer-alt-framework git branch 'testnet' (moving target) — pin/lockfile matters if the team vendors or rebuilds it.
- If older docs/configs surface other package IDs, ignore them; only 0xf5ea2b...5138 is current testnet, and the indexer only matches events from configured package addresses (upgrades require --predict-package-id override).
