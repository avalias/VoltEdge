# moveCore

## Summary
Mapped the full DeepBook Predict Move package (C:/Users/tvoiv/Desktop/SuiOverflowVal/vendor/deepbookv3-predict/packages/predict/sources/, module namespace `deepbook_predict`, depends on local `deepbook` package). Core model: one shared OracleSVI per (underlying, expiry) holding spot/forward + raw SVI total-variance params pushed by a Block Scholes operator cap; binary UP price = N(d2) where w(k)=a+b(rho(k-m)+sqrt((k-m)^2+sigma^2)) is TOTAL variance (no time scaling on-chain — ms_per_year constant exists but is unused; the operator bakes T into a,b). All prices/percentages are fixed-point 1e9 (float_scaling), quantities are quote units with required 6 decimals (1_000_000 = 1 contract = $1 payout). predict.move is the orchestrator: mint/redeem/mint_range/redeem_range against a shared PredictManager (wraps a DeepBook BalanceManager + position Tables keyed by MarketKey/RangeKey), and PLP supply/withdraw against a single Vault that takes the other side of every trade. Quoting: bid/ask = N(d2) ∓ spread where spread = max(base_spread·sqrt(p(1-p)), min_spread) + base_spread·utilization_multiplier·(min(mtm/balance,1))²; DN side is parity-mirrored (dn_ask=1-up_bid). Quotes are computed POST-trade (mint inserts exposure first, then quotes). Vault tracks per-oracle dense StrikeMatrix (512-slot pages + prefix-summary tree) giving exact max_payout (worst-case settled payout) and curve-sampled MTM (adaptive ≤50-point piecewise-linear curve of N(d2)); vault_value = balance − total_mtm; LP shares = amount·total_supply/vault_value rounded down; withdraw limited to balance − total_max_payout and gated by a Chainlink-CCIP-style token-bucket RateLimiter. Settlement: first update_prices at/after expiry freezes settlement_price = pushed spot, then compute_price returns 1e9/0 digital payoff, redeem_permissionless becomes available, and compact_settled_oracle collapses the matrix to (remaining_quantity, remaining_liability). NOTE: there are NO functions named mint_collateralized/redeem_collateralized/get_quote in this codebase — the quoting entry points are get_trade_amounts/get_range_trade_amounts and the internal trade_prices/range_trade_prices; ranges (vertical spreads) are the second instrument class. ### SVI parameterization
struct SVIParams has copy,drop,store { a: u64 (overall variance level >=0), b: u64 (wing slope >=0), rho: i64::I64 (signed skew, typically negative), m: i64::I64 (signed horizontal shift), sigma: u64 (ATM curvature >=0) } — all scaled by 1e9. This is raw SVI in TOTAL VARIANCE form: w(k) = a + b*(rho*(k-m) + sqrt((k-m)^2 + sigma^2)). There is NO theta/annualization on-chain; a and b already embed time-to-expiry (operator-side responsibility). struct PriceData { spot: u64, forward: u64 } also 1e9-scaled, updated ~1s; SVI updated ~10-20s. i64::I64 = { magnitude: u64, is_negative: bool } with normalized zero (from_parts forces is_negative=false when magnitude==0); ops: add/sub/neg/mul_scaled (a*b/1e9)/div_scaled (a*1e9/b)/square_scaled (returns u64 magnitude).

### binary price formula (compute_nd2)
fun compute_nd2(oracle,&OracleSVI, strike:u64): u64 — exact steps: (1) forward=oracle.prices.forward, abort EZeroForward if 0; (2) k = predict_math::ln(math::div(strike, forward)) — signed I64 log-moneyness ln(K/F); (3) k_minus_m = k - m; (4) total_var = a + mul(b, |rho*(k-m) + sqrt((k-m)^2*1e-9... i.e. square_scaled(k_minus_m) + mul(sigma,sigma) then sqrt(.,1e9)|); aborts ECannotBeNegative if rho*(k-m)+sqrt(...) < 0, EZeroVariance if total_var==0; (5) d2 = -((k + total_var/2) / sqrt(total_var)) via i64 div_scaled; (6) return predict_math::normal_cdf(&d2). So UP price = N(d2) with d2 = -(ln(K/F) + w/2)/sqrt(w), w = SVI total variance at k. This equals P(S_T > K) under lognormal forward measure. binary_price_pair(oracle, strike, _clock) returns (up, 1e9 - up): exact parity UP+DN=1. compute_price(oracle, strike) is public(package): if settled returns 1e9 if settlement_price > strike else 0 (strict >, so DN wins ties at strike==settlement); else compute_nd2.

### bid/ask spread formula (exact)
fun trade_prices(...): up_price = oracle.compute_price(strike). If settled: ask=bid=fair (UP price or 1e9-up for DN). Else spread = pricing_config.quote_spread_from_fair_price(up_price, vault.total_mtm() [liability], vault.balance()): spread = max(mul(base_spread, sqrt(mul(p,1e9-p),1e9)), min_spread) + utilization_spread, where utilization_spread = 0 if balance==0||liability==0, else util=min(div(liability,balance),1e9); util_sq=mul(util,util); mul(base_spread, mul(utilization_multiplier, util_sq)). I.e. spread = max(base_spread·√(p(1−p)), min_spread) + base_spread·util_mult·util². Then up_bid = max(up_price−spread, 0); up_ask = min(up_price+spread, 1e9); dn_bid = 1e9−up_ask; dn_ask = 1e9−up_bid (parity-consistent). Aborts EFairPriceAlreadySettled if fair price is exactly 0 or 1e9 while unsettled. Range version: fair = up(lower) − up(higher) (monotone non-increasing), same Bernoulli+utilization spread on the range fair price, ask=min(fair+spread,1e9), bid=max(fair−spread,0). IMPORTANT for the strategy: the spread uses post-trade total_mtm/balance because mint/redeem mutate exposure before quoting; preview via get_trade_amounts uses CURRENT state, so executed price can differ from preview (the trade's own MTM impact).

### StrikeMatrix data structure

## Gotchas
- No get_quote / mint_collateralized / redeem_collateralized functions exist — the prompt's assumed API differs. Quoting = get_trade_amounts / get_range_trade_amounts / ask_bounds; trading = mint / redeem / redeem_permissionless / mint_range / redeem_range; LP = supply / withdraw. Plan the PTB layer around these.
- SVI params are TOTAL variance, not annualized vol: w(k) = a + b(rho(k−m) + sqrt((k−m)²+sigma²)) is used directly in d2 with no multiplication by time-to-expiry. ms_per_year (31_536_000_000) is defined in constants.move but referenced NOWHERE. To plot implied vol on the surface viewer compute sigma_iv = sqrt(w(k)/T) off-chain with T = (expiry − now)/ms_per_year.
- Quotes are POST-TRADE: mint inserts the position into the vault matrix and refreshes MTM BEFORE computing the ask, so the executed price includes the trade's own utilization/MTM impact and will be worse than a get_trade_amounts preview taken pre-trade. Same (mirrored) for redeem. The ladder bot must model this slippage or re-quote inside the same PTB.
- Spread asymmetry across UP/DN: spread is computed from the UP price's Bernoulli variance and then mirrored (dn_ask = 1e9 − up_bid), so UP and DN quotes share one spread — parity ask_up + bid_dn = 1e9 + 2·spread... actually ask_up + ask_dn = 1e9 + 2·spread and bid_up + bid_dn = 1e9 − 2·spread. A no-arb checker should verify ask_up+ask_dn >= 1e9 >= bid_up+bid_dn, monotonicity of N(d2) in strike, and (for ranges) up(lower) − up(higher) >= 0.
- quote_spread_from_fair_price ABORTS (EFairPriceAlreadySettled) when the live fair price is exactly 0 or 1e9 — deep ITM/OTM strikes where N(d2) saturates make trade_prices/get_trade_amounts abort even though the oracle is live. The surface viewer's devInspect calls must handle this abort, and the ladder must avoid saturated strikes (also blocked by min/max ask bounds 1%/99% anyway).
- Oracle staleness threshold is 30s from oracle.timestamp, and timestamp is bumped ONLY by update_prices (not update_svi). PENDING_SETTLEMENT (post-expiry, pre-settlement-push) freezes ALL quoting and redeeming until the operator pushes one more price. Settlement price = the SPOT field of the first update_prices at/after expiry; UP wins on settlement > strike STRICTLY (ties go to DOWN); range pays on (lower, higher] half-open.
- Vault has TWO different solvency lenses: mint risk gate uses total_mtm <= 80% of balance, but LP withdraw availability uses balance − total_max_payout (worst-case payout from the exact prefix-max over the strike book) — much stricter. Monte-Carlo dashboard should track both: vault_value = balance − total_mtm (share pricing) and free = balance − total_max_payout (withdrawability).
- Withdrawal token-bucket starts DISABLED (returns u64::MAX available) and capacity 0; on testnet it may or may not be enabled — read state before modeling. record_deposit replenishes the bucket on supply.
- No fees anywhere: no protocol fee, no fee ledger; LP PnL = spread capture + settled-loser premiums retained in vault.balance. PLP share math: supply shares = floor(amount·total_supply/vault_value) (pre-deposit vault_value), withdraw amount = floor(shares·vault_value/total_supply), full-supply burn gets exact vault_value.
- MTM curve quirk: refresh_oracle_risk builds the curve over the HISTORICAL minted strike range (minted bounds never shrink even after full unwind), with up to 50 adaptive samples bisecting the largest UP-price gaps and snapping midpoints down to ticks. The on-chain MTM is therefore an interpolation, not exact — a vault-risk dashboard replicating evaluate() must implement the segment semantics: UP counted in (k_i, k_{i+1}], DN in [k_i, k_{i+1}), endpoints exact, price interpolated at the inventory's average strike.
- redeem (non-permissionless), mint, deposit, withdraw on PredictManager are owner-gated by ctx.sender() == manager.owner addr — the bot wallet must be the manager creator. PredictManager is a SHARED object (contention: all of the team's trades through one manager serialize on it, as do all trades globally on the shared Predict + OracleSVI objects).
- Strikes/spot/forward/settlement all share the 1e9 fixed-point price scale; quantities and costs are 6-decimal quote units; cost = mul(ask_1e9, quantity_6dec) mixes scales correctly because mul divides by 1e9. Strike grids are huge: 100_000 ticks of tick_size (multiple of 10_000 = $0.00001 at 1e9 scale... i.e. 1e4/1e9), max_strike = min_strike + tick_size·100_000.
