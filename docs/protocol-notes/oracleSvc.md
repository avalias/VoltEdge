### On-chain SVI pricing math (for no-arb checker)
oracle.move compute_nd2: k = ln(strike/forward) (I64); w(k) = a + b*(rho*(k−m) + sqrt((k−m)² + σ²)); asserts inner = rho*(k−m)+sqrt(...) is non-negative (ECannotBeNegative=4) and total_var > 0 (EZeroVariance=5), forward > 0 (EZeroForward=3); d2 = −((k + w/2)/sqrt(w)); UP price = normal_cdf(d2). binary_price_pair returns (up, float_scaling − up) — UP+DN=1 exact parity. Settled oracle: UP = 1e9 iff settlement_price > strike (STRICTLY greater), else 0. Errors: EInvalidOracleCap=0, EOracleAlreadyActive=1, EOracleExpired=2, EZeroForward=3, ECannotBeNegative=4, EZeroVariance=5, EOracleSettled=6. Status codes u8: INACTIVE=0, ACTIVE=1, PENDING_SETTLEMENT=2, SETTLED=3.

## Gotchas
- The three files named in the task spec (blockscholes-oracle.ts, oracle-feed.ts, oracle-dashboard.ts) are phantom names from PREDICT_MIGRATION.md's PR-10 plan — the dashboard was never implemented in this repo. Don't go looking for its data-access patterns; the public predict-server REST API + the feed's devInspect/getDynamicFields patterns are the real reference.
- WS subscription decimals MUST be >=9 (WS_SUBSCRIPTION_DECIMALS=9): SVI params are scaled 1e9 on-chain, and 5-decimal formatting quantizes short-dated `a` to zero — a previously-shipped bug documented in eng-rules/scripts.md. If your terminal re-derives surfaces from BlockScholes directly, keep full precision.
- SVI a, b, sigma are u64 on-chain — the protocol cannot represent negative `a` (raw SVI fits sometimes produce a<0 for very short expiries). The feed would crash/abort on tx.pure.u64 of a negative scaled value; your no-arbitrage checker should treat a>=0, b>=0, sigma>=0 as protocol invariants, plus the on-chain asserts: rho*(k−m)+sqrt((k−m)²+σ²) >= 0 and total_var > 0.
- Settlement price source is BlockScholes index.px spot pushed by the feed operator — not an exchange settlement print. UP wins on settlement_price > strike STRICTLY (ties pay DOWN). update_prices' forward arg is ignored in the settlement branch.
- oracle.timestamp is bumped only by update_prices, never by update_svi; on-chain staleness gate is 30s (staleness_threshold_ms). The SVI params themselves can be up to ~20s+ old while the oracle is 'fresh'. Monitor both: prices age vs 30s, and OracleSVIUpdated age vs ~40s (2 missed retransmits).
- Oracle identity is (underlying, expiry), NOT (tier, expiry): 15m and 1h schedules overlap on the hour (and 1d/1w at 08:00 UTC), and inferTier assigns each oracle to only one tier. Dedupe/grouping in your terminal must key by expiry alone or you'll double-count.
- MIN_LOOKAHEAD_MS=1h applies only at CREATION: the '15m tier' never has a brand-new oracle expiring <1h out, but the steady-state ladder includes oracles expiring every 15min as old ones roll down. Don't assume nearest expiry >= 1h.
- The feed stops pushing prices 5s before expiry (SAFETY_WINDOW_MS) and on-chain rejects mint/redeem in the PENDING_SETTLEMENT gap (assert_quoteable_oracle). Your range-ladder must stop quoting/minting before expiry−5s and expect a dead window of up to one manager interval (default 5min, usually seconds since shouldRunManagerWindowNow fires on the next 1s tick) before settlement lands.
- scaleToU64 uses Math.round(value*1e9) in JS doubles — values from the indexer (i64 1e9-scaled) and your own re-computation can differ by 1 ULP; never bit-compare, use tolerance.
- Indexer /oracles status enum is 'created'|'active'|'settled' (no pending_settlement/compacted) and is event-derived; feed-side status has 5 states. For real-time state (pending settlement gap) use /oracles/:id/state latest_price timestamps or direct RPC on the OracleSVI object.
- Vendored constants.ts IDs may be stale relative to the live predict-testnet-4-16 deployment behind predict-server.testnet.mystenlabs.com — confirm package/object ids via GET /oracles and /config before hardcoding into the terminal.
- Vault MTM fields (total_mtm, total_max_payout) only exist on the live Predict object, not in the indexer DB — vault risk dashboard needs a Sui RPC getObject(predict_id) poller like fetch_predict_snapshot, plus event history for the performance series.
- BlockScholes mark.px rejects past/near-term expiries and one bad sid poisons the whole 20-sub batch — if you subscribe directly, filter expiries and chunk to 20 with stable client_ids (re-sending a client_id replaces, not appends, its subs).
- Move package events: rho and m inside OracleSVIUpdated parse as nested I64 structs ({magnitude, is_negative}) in parsedJson — not plain integers; the indexer flattens them to (rho, rho_negative) columns.
